Chromatic Cipher
Multi-Layer Visual Steganographic Encryption
🔒
Encrypt
Hide a secret message inside an image using double encryption and a colour-coded steganographic pixel map.
🔓
Decrypt
Recover the hidden message from an encoded image using your encrypted keyset and passphrase.

Step 1 — Double Encryption: Your message is first encrypted with AES-256-GCM (random IV), then the output is re-encrypted with XChaCha20-Poly1305, producing a ciphertext byte array.

Step 2 — Colour Mapping: Every unique byte value (0–255) in the ciphertext is assigned 20 globally-unique random RGB colours. No two bytes share any colour triple across the entire map.

Step 3 — Feistel Pixel Placement: A 4-round Feistel cipher (seeded from the AES key via SHA-256) generates a deterministic permutation of all pixel positions, scattering encrypted pixels unpredictably across the image.

Step 4 — Noise Fill: Remaining pixel positions are filled with random RGB noise, making the image look like pure static.

Step 5 — Keyset Encryption: All keys and the colour map are bundled into JSON, encrypted via Argon2id-derived AES-GCM (memory-hard, m=8192 KB, t=2), then the derived key is ECDH-wrapped (ephemeral X25519 + HKDF + AES-GCM). The result is a single JSON bundle safe to share.

Decryption reverses every step using the RSA private key (held in session memory), the passphrase, and the uploaded image.

❓  How To Use This App
What is this?

Chromatic Cipher hides a secret message inside a normal-looking image. The message is encrypted, converted into coloured pixels, and scattered across the image so it looks like random noise. Only someone with the correct keyset file, passphrase, and private key can recover the original message.

📤 Scenario A — Sending a secret message to someone
1
Recipient opens the app first
The person receiving your message opens Chromatic Cipher. Their public key (44 characters) is automatically generated on Page 1. They copy it and send it to you.
2
You open the app and go to Encrypt
On Page 1 you see your own keys — ignore them for now. Click Continue, then choose Encrypt.
3
Fill in the encrypt form
• Message — type your secret message
• Recipient's Public Key — paste the 44-char key they sent you
• Image size — choose a size (512×512 is fine for short messages)
• Colours per character — leave at 20 unless you need more security
• Passphrase — choose a strong password, share it with the recipient separately (not in the same channel)
4
Download both output files
After encryption, download the PNG image and the Bundle JSON. Send both files to the recipient. Send the passphrase separately (e.g. by phone or in person).
5
Recipient decrypts
Recipient is still in their session (or pastes their private key). They go to Decrypt, upload the PNG, paste the Bundle JSON, enter the passphrase — and the message appears.
🔐 Scenario B — Encrypting for yourself
1
Open the app and save your private key
On Page 1, click Copy Private Key and save it somewhere safe. You will need it if you close the browser.
2
Encrypt your message
Go to Encrypt. Leave the Recipient Public Key field empty — it will use your own session key automatically. Fill in your message and a passphrase.
3
To decrypt later
Open the app, go to Decrypt, upload the PNG and Bundle JSON. If you are in the same session leave the private key field empty. If you reopened the browser, paste your saved private key into the optional field.
📋 Quick Reference — What each file does
🖼
chromatic_cipher.png
The image carrying your hidden message. Looks like noise. Safe to share publicly.
📦
chromatic_cipher_bundle.json
The encrypted keyset. Contains all the keys needed to decode the image — but they are encrypted. Useless without the passphrase and private key. Share with the recipient.
🔑
chromatic_cipher_private_key.txt
Your private key — 44 characters. Keep it secret and safe. Only needed if you close and reopen the browser. Never share it.
⚠ Important reminders:
— Never send the passphrase in the same message as the image + bundle
— The private key must stay secret — it unlocks everything
— Once you close the browser your session keys are gone — save your private key first
— Always use a strong passphrase — the security of the system depends on it
